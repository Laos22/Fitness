Компоненты хедера должны соответствовать следующим стандартам:

1. **Структура и вёрстка**:
   - Использовать семантический элемент `<header>` с соответствующими ARIA-атрибутами
   - Реализовывать адаптивный дизайн с подходом mobile-first
   - Использовать flexbox или CSS Grid для выравнивания элементов
   - Обеспечивать достаточные отступы и padding для touch-элементов (минимум 44px в высоту)

2. **Логотип/Бренд**:
   - Размещать логотип в крайнем левом положении (или по центру для мобильных)
   - Использовать семантический `<div>` или `<h1>` для текста логотипа
   - Добавлять соответствующий alt-текст для изображений логотипа
   - Обеспечивать переход на главную страницу по клику на логотип

3. **Навигация**:
   - Использовать семантический элемент `<nav>`
   - Создавать доступную навигацию с правильным управлением фокусом
   - Реализовывать выпадающие меню для сложной навигации
   - Добавлять индикаторы активного состояния для текущей страницы
   - Включать эффекты при наведении и плавные переходы

4. **Поиск**:
   - Размещать элемент поиска в правой части или по центру
   - Использовать правильные элементы формы с обработчиками отправки
   - Реализовывать клавиатурную навигацию (Enter для отправки)
   - Добавлять состояния загрузки для результатов поиска
   - **В мобильной версии при открытом меню элемент поиска должен скрываться**

5. **Аутентификация**:
   - Размещать кнопки входа/регистрации в правой части
   - Отображать аватар/меню пользователя при авторизации
   - Использовать согласованные стили кнопок и цвета
   - Обрабатывать изменения состояния аутентификации

6. **Доступность**:
   - Добавлять соответствующие ARIA-метки и роли
   - Обеспечивать работу клавиатурной навигации
   - Использовать семантические HTML-элементы
   - Добавлять индикаторы фокуса и ссылки для пропуска
   - Поддерживать экраны чтения с соответствующими метками

7. **Адаптивный дизайн**:
   - Скрывать/показывать элементы в зависимости от размера экрана
   - Реализовывать мобильное меню с гамбургер-иконкой
   - Использовать сворачиваемую навигацию на маленьких экранах
   - Поддерживать удобные для касания взаимодействия
   - **Элементы поиска и аутентификации в мобильной версии переносятся в мобильное меню**

8. **Производительность**:
   - Оптимизировать изображения и иконки
   - Использовать эффективные CSS-in-JS или CSS Modules
   - Минимизировать повторные рендеры с помощью memoization
   - Lazy load тяжелые компоненты

9. **Стилизация**:
   - Использовать согласованную цветовую палитру и типографику
   - Реализовывать правильный z-index для перекрывающихся элементов
   - Добавлять тонкие тени и границы для глубины
   - Поддерживать темный режим, если необходимо

10. **Управление состоянием**:
    - Управлять состоянием мобильного меню локально
    - Обрабатывать состояние аутентификации глобально
    - Использовать контекст или state management для темы
    - Реализовывать правильную очистку при unmount
